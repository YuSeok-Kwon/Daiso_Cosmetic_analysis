## 2. 변경 내용

### 2.1 "색상/발색" Aspect 신규 추가

**변경 이유:**

- 기존 "디자인" aspect의 75%가 실제로는 제품 색상/발색에 대한 리뷰
- 실제 패키징 디자인은 10%에 불과

**분류 기준 (키워드):**

```
색상/발색: 색, 컬러, 톤, 핑크, 레드, 브라운, 코랄, 누드,
          발색, 쿨톤, 웜톤, MLBB 등
디자인: 포장, 패키지, 케이스, 용기, 예쁘다, 귀엽다 등
```

### 2.2 "사이즈/호환" → "용량/휴대" 명칭 변경

**변경 이유:**

- 화장품 리뷰 특성상 "사이즈"보다 "용량"이 더 적합
- "호환"보다 "휴대성"이 더 자주 언급됨

### 2.3 "품질/불량" → "품질" 명칭 변경 및 정의 명확화

**변경 이유:**

- "불량"이라는 표현이 너무 한정적
- 품질 = 물리적·제조적·유통적 결함/하자 여부

**품질 정의:**

```
품질(긍정): 제품이 물리적·제조적·유통적으로 정상이고 신뢰할 수 있다는 인식
  예: "불량 걱정했는데 이상 없이 잘 왔어요", "깨짐이나 새는 거 없이 정상 제품이에요"

품질(부정): 제품이 '정상 상태'가 아님 (결함/하자/이상)
  예: "불량이에요", "변질됐어요", "유통기한 지났어요"

주의: 배송 중 파손은 "배송/포장"으로 분류
```

## 3. 직접검증 수행

### 3.1 검증 방법

- `aspect_evaluation_guide.md` 규칙 기반 검증
- 정규식 패턴 매칭을 통한 aspect 검출
- 오분류 패턴 수정 및 누락 aspect 추가
- sentiment_score 극단화 조정

### 3.2 주요 수정 규칙

| 규칙 | 설명 |
|------|------|
| 배송 중 파손 | 품질 → 배송/포장 |
| "재구매" 태그만 | 재구매 aspect 제외 (실제 의사 없음) |
| "그냥 그래요" | neutral → 약한 negative (-0.3) |
| 강한 부정 표현 | sentiment_score = -1.0 |
| 강한 긍정 표현 | sentiment_score = 1.0 |
