2. Soft Hierarchy 설계 원칙 (지금 상황에 최적)

* 모델 출력은 10개 ASPECT 그대로
* hierarchy는 후처리/분석/대시보드용
* 중복 매핑 허용 (soft니까)
* annotator 재교육 없음
* 재라벨링 없음

Soft Hierarchy 상위 ASPECT (한국어 버전)

제품 경험 (PRODUCT_EXPERIENCE)

1. 사용 체감

실제 사용하면서 느끼는 전반적인 느낌

* 사용감/성능
* 재질/냄새

의미: 바를 때 느낌, 흡수, 촉감, 냄새 등 사용 중 체감 요소

2. 외관 표현 성능

보이는 결과와 시각적 만족

* 색상/발색
* 디자인

의미: 메이크업 결과, 컬러 만족도, 패키지 외관 등 시각적 평가

3. 물리적 가치

제품의 물리적 구성과 신뢰도

* 용량/휴대
* 품질/불량

의미: 양, 휴대성, 내구성, 하자 여부 등 제품 자체 완성도

4. 가격 가치

가격 대비 만족도

* 가격/가성비

의미: 저렴함, 가성비, 가격 대비 성능 인식

5. 재구매 의도

향후 구매 행동에 대한 의사

* 재구매

의미: 다시 살 의향, 추천, 지속 사용 의도

구매 경험 (PURCHASE_EXPERIENCE)

6. 배송 유통 경험

배송 과정 및 유통 품질

* 배송/포장

의미: 배송 속도, 포장 상태, 파손 여부

7. 고객 응대 경험

구매 이후의 서비스 대응

* CS/응대

의미: 교환 환불, 고객센터 응대, 문제 처리 경험

한국어 매핑 테이블 (바로 사용 가능)

SOFT_HIERARCHY_KR = {
# 제품 경험
"사용 체감": ["사용감/성능", "재질/냄새"],
"외관·표현 성능": ["색상/발색", "디자인"],
"물리적 가치": ["용량/휴대", "품질/불량"],
"가격 가치": ["가격/가성비"],
"재구매 의도": ["재구매"],

```
# 구매 경험
"배송·유통 경험": ["배송/포장"],
"고객 응대 경험": ["CS/응대"]

```

}

Inference 후처리 예시 (한국어)

Leaf ASPECT 출력

"가격/가성비": 0.74,
"배송/포장": 0.12
}

상위 ASPECT 집계

{
"사용 체감": 0.82,
"가격 가치": 0.74,
"배송·유통 경험": 0.12
}

(집계 방식: max 추천)

6. 이 구조의 실전 장점

지금 당장

* 대시보드에서
* “제품 문제 vs 구매 문제” 명확 분리
* 브랜드/제품 비교 쉬움

중기

* 상위 ASPECT 기준 KPI 가능
* 리뷰 요약 자동화 쉬움

장기

* 나중에 hierarchical model로 넘어가도
-> 이 매핑이 그대로 supervision 힌트

7. 꼭 추가하면 좋은 옵션 (선택)

중복 매핑 (soft의 장점 활용)

예:

"product_physical_value": ["용량/휴대", "품질/불량"],
"product_usage_experience": ["사용감/성능", "재질/냄새"]

-> 품질/불량을 usage에도 간접 영향으로 해석 가능
