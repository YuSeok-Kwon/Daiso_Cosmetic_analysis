## 데이터 분석 프로젝트 Todo List

---

### 0. 문제 정의

**목적:** 데이터를 만지기 전에 "무엇을, 왜 해결하려 하는가?" 정의

- [ ] 분석 목적 및 배경 설정: 왜 이 분석이 필요한가? (비즈니스 관점)
- [ ] 이해관계자 정의: 분석 결과를 누가 소비하는가? (의사결정자, 실무자 등)
- [ ] 가설 설정: 어떤 결과가 나올 것이라고 예상하는가?
- [ ] 평가 지표(Metric) 설정: 무엇을 기준으로 성공/실패를 판단할 것인가?
- [ ] 분석 범위 및 제약 조건 정의: 시간, 데이터, 리소스의 한계 명시

---

### 1. 팀원 및 이해관계자 싱크

**목적:** 팀원 및 이해관계자와의 방향성 합의

- [ ] 킥오프 미팅: 분석 목적, 기대 결과물, 일정 합의
- [ ] 역할 분담: 팀원별 담당 영역 및 책임 명확화
- [ ] 커뮤니케이션 채널 확정: 슬랙, 노션, 깃허브 등 협업 도구 선정
- [ ] 마일스톤 설정: 중간 점검 일정 및 최종 발표 일정 확정

---

### 2. 데이터 수집

**목적:** 분석에 필요한 원천 데이터 확보

- [ ] 데이터 소스 확보: 크롤링, Open API, CSV 파일 등
- [ ] 데이터 로드: Pandas 등을 통한 데이터 불러오기
- [ ] 데이터 명세서 작성: 컬럼별 의미, 데이터 타입, 단위 등 정리
- [ ] 데이터 품질 검증: 수집 시점의 정합성 체크 (누락, 중복, 범위 이상 등)

---

### 3. 데이터 전처리

**목적:** EDA를 위한 데이터 정제

- [ ] 결측치 처리: 삭제 vs 보간(Fillna, 평균/중앙값/최빈값 등)
- [ ] 이상치 탐지 및 처리: IQR, Z-Score 등 활용
- [ ] 데이터 타입 변환: 날짜(Date), 수치형(Int/Float), 범주형(Category) 등
- [ ] 중복 데이터 제거
- [ ] 전처리 로직 함수화: 재사용 및 파이프라인 구축 대비

---

### 4. EDA

**목적:** 데이터 패턴 파악 및 인사이트 초안 도출

- [ ] 기술 통계량 확인: `describe()`, `info()`, 상관계수 확인
- [ ] 단변량 분석: 개별 컬럼의 분포 확인 (Histogram, Boxplot, Countplot)
- [ ] 다변량 분석: 변수 간의 관계 확인 (Scatter plot, Heatmap, Groupby 집계)
- [ ] 가설 검증(1차): 시각화와 통계를 통해 초기 가설 검토
- [ ] 중간 점검: EDA 결과 공유 및 이해관계자 피드백 수렴

---

### 5. 피처 엔지니어링

**목적:** 모델 성능 향상을 위한 변수 재가공

- [ ] 파생 변수 생성: 기존 변수 조합
- [ ] 피처 선택: 상관관계, VIF, 변수 중요도 기반 선별
- [ ] 스케일링: 표준화, 정규화
- [ ] 인코딩: 원-핫 인코딩, 레이블 인코딩
- [ ] 데이터셋 분리: Train / Validation / Test Set 분리

### 6. 모델링 및 학습

**목적:** 문제 해결을 위한 모델 구축

- [ ] 베이스라인 모델 수립: 가장 단순한 모델로 기준점 설정
- [ ] 모델 선택 및 학습: 문제 유형(분류/회귀/추천)에 맞는 알고리즘 적용
- [ ] 교차 검증: K-Fold 등으로 모델 안정성 확인
- [ ] 하이퍼파라미터 튜닝: GridSearch, RandomSearch, Optuna 등 활용

### 7. 모델 검증 및 평가

**목적:** 모델 성능 검증 및 개선점 도출

- [ ] 성능 평가: 설정한 지표(Accuracy, F1-score, RMSE, AUC 등)로 평가
- [ ] 과적합 확인: Train 점수와 Test 점수의 차이 비교
- [ ] 모델 해석: SHAP, Feature Importance 분석
- [ ] 오답 분석: 모델이 무엇을 틀렸는지, 왜 틀렸는지 확인
